Phần 03 - Tạo Facebook Data Analysis Report¶

Welcome bạn đến phần 3 của bài Lab Facebook Social Analysis Report của Microsoft Student Partners Vietnam nhé. Bài Lab này do MSP Lại Trung Minh Đức (Technical Trainer) trình bày và hướng dẫn.

Trong phần 3 này, bạn sẽ dùng Python để khai thác và phân tích dữ liệu Facebook của cá nhân bạn ở phần 2. Phần phân tích bao gồm:

  • Thống kê mô tả Feeds, Comments và Reactions để đánh giá những cái "nhất".
  • Tìm các n-TOP của Feeds, Comments, và Reactions
  • Tạo WordCloud cho Feeds (3 WordCloud-s)
  • Tạo Chart cho Comments theo thời gian (8 Chart-s)

Để thực hành phần Lab này, bạn cần có

  • User ID từ Facebook Graph API
  • MongoDB URL và Database name đã có dữ liệu ở phần 2

Bạn cũng sẽ cần cài đặt packages:

  • WordCloud

Let's get started

3. Thống kê mô tả dữ liệu với DataFrame.describe()¶

  • Đưa ra bảng thống kê mô tả (Descriptive Statistical) của dữ liệu.
  • Nếu dữ liệu là số (numeric) --> Bảng thống kê sẽ có: count, mean, std, min, max,...
  • Nếu dữ liệu là chữ (string) --> Bảng thống kê sẽ có: count, unique, top, freq

Một lưu ý khi dùng describe:

  • To call describe() on just the numerical columns use describe(include = [np.number])
  • To call describe() on just the objects (strings) using describe(include = ['O'])
  • Ref: https://stackoverflow.com/questions/24524104/pandas-describe-is-not-returning-summary-of-all-columns

5.Tìm n-top có có group với DataFrame.groupby(columns_list)¶

  • Mục tiêu của hàm này là để gom nhóm data giống nhau theo cột (hoặc nhiều cột)
  • Thường hàm sẽ được dùng với một số hàm khác theo chaining để tính toán, hoặc tạo ra cột mới.

Quay trở lại data Comments và Reactions, ta thấy có thể gom nhóm 2 cột user_id và cột user_name. Do đó, ta dùng codeblock dưới đây để gom nhóm --> tạo cột mới tên counts --> dùng cột counts này để tìm n-top.

Reference: https://stackoverflow.com/questions/19384532/how-to-count-number-of-rows-in-a-group-in-pandas-group-by-object

6. Tạo WordCloud¶

  • Định nghĩa: WordCloud là một tấm hình - được tạo từ sự sắp xếp rất nhiều từ, cụm từ - Kích thước của từng từ sẽ dựa trên độ quan trọng / tần suất xuất hiện của từ.
  • Mục đích: Nắm nhanh qua nội dung các Feeds/Comments thông qua WordCloud

Quy trình tạo WordCloud đơn giản¶

  • Đưa nội dung cần tạo WordCloud vào 1 biến String (nội dung sách, nội dung messages,...).
  • Import các package cần thiết của WordCloud.
  • Import các STOPWORDS package tuỳ theo từng loại ngôn ngữ.
  • Chạy WordCloud
  • Xem Result rồi điều chỉnh lại thông số cho hợp lý
  • Lưu WordCloud result vào file ảnh

Nếu bạn thắc mắc STOPWORDS là gì thì theo mình hiểu, đó là những từ đệm được thêm vào văn bản để làm cho bài văn smooth hơn, mềm mại hơn. Thường các từ này chiếm khá nhiều chỗ trong bài của bạn nên bạn có thể bỏ nó đi. Bạn có thể xem thêm tại: https://en.wikipedia.org/wiki/Stop_words

STOPWORDS tiếng Anh mình sẽ dùng default của WordCloud STOPWORDS tiếng Việt mình sẽ tự parse file tại http://xltiengviet.wikia.com/wiki/Danh_s%C3%A1ch_stop_word

Guide config WordCloud: https://amueller.github.io/word_cloud/generated/wordcloud.WordCloud.html

Phía trên là quy trình đơn giản, bây giờ là cách làm của mình dành cho việc tạo WordCloud cho nội dung Feeds của mình.

7. Tạo Line Chart bằng Matplotlib với plt.plot()¶

  • Trong Python có khá nhiều thư viện hỗ trợ để dựng Chart như Matplotlib, Plot.ly,... và hiện tại để cho đơn giản, mình sẽ dùng Matplotlib
  • Muốn dùng được thì ta sẽ cần phải import thư viện vào (và mình đã import ở phía trên)
  • Đây là link tutorial của riêng Matplotlib: https://matplotlib.org/tutorials/index.html. Nếu bạn muốn học cách sử dụng Matplotlib thì có thể xem ở đây nhé.

  • Line chart (với plt.plot())
  • Scatter (với plt.scatter())
